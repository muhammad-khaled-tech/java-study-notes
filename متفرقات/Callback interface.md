### سؤال: يعني إيه كلاس `Timer` وبيعمل إيه في حياتنا؟

#### س1: تخيل إنك بتسلق بيض.. بتعمل إيه؟

ج:

أنت بتحط البيض في النار، وبتظبط "المنبه" (Timer) بتاع المطبخ على 5 دقايق.

المنبه ده جهاز غبي، وظيفته حاجة واحدة بس:

1. يعد 5 دقايق.
    
2. أول ما يخلصوا، يعمل صوت "ترررررن" (Event).
    
3. أنت لما بتسمع الصوت، بتجري تشيل البيض (Action).
    

هو ده بالظبط كلاس Timer في الجافا!

هو كلاس جاهز موجود في مكتبة اسمها javax.swing، وظيفته إنه يشتغل كـ "عداد وقت". بتقوله: "عد كذا ثانية، ولما تخلص، نادي عليا عشان أنفذ كود معين".

---

#### س2: الكلاس ده محتاج مني إيه عشان يشتغل؟

ج:

عشان تشتري منبه وتشغله، محتاج تظبط حاجتين:

1. **المدة (Delay):** هيرن كل قد إيه؟ (مثلاً كل 1000 مللي ثانية = 1 ثانية).
    
2. **المستمع (Listener):** مين الشخص اللي هيرد عليه لما يرن؟
    
    - المشكلة إن التايمر ده "أعمى". هو ميعرفش ينادي على "محمد" ولا "أحمد".
        
    - هو مبرمج إنه يدوس على "زرار" معين لما الوقت يخلص.
        

---

#### س3: إيه حكاية "الزرار" أو الإنترفيس `ActionListener`؟

ج:

تخيل التايمر ده علبة مقفولة طالع منها سلك. السلك ده لازم يتركب في "فيشة" معينة.

الفيشة دي هي الـ Interface اللي اسمه ActionListener.

- **التايمر بيقولك:** "يا عم المبرمج، أنا لما الوقت يخلص، ببعت إشارة كهرباء في السلك ده. هاتلي بقى أي **كلاس** (فيشة) بيفهم الإشارة دي وركبه فيا".
    
- **أنت بتعمل إيه؟** بتعمل كلاس (TimePrinter) وتخليه يركب الفيشة دي (`implements ActionListener`).
    
- **النتيجة:** لما التايمر يخلص عد، الشحنة بتمشي وتوصل للكلاس بتاعك، فتشغل الدالة اللي جواه (`actionPerformed`).
    

---

#### س4: إزاي بنكتب الكلام ده كود (ببساطة شديدة)؟

ج:

تعالى نترجم حدوتة البيض لكود جافا:

1. نجهز الشخص اللي هيشيل البيض (المستمع):
    
    لازم يكون معاه "رخصة" (Interface) إنه بيسمع المنبه.
    
    Java
    
    ```
    // ده الشخص اللي هيتحرك لما المنبه يرن
    class Person implements ActionListener {
        @Override
        public void actionPerformed(ActionEvent e) {
            System.out.println("أنا شيلت البيض من على النار!"); 
        }
    }
    ```
    
2. **نشتري المنبه ونظبطه:**
    
    Java
    
    ```
    // 1. جبنا الشخص
    ActionListener mom = new Person();
    
    // 2. جبنا المنبه (Timer)
    // وقلنا له: رن كل 5000 مللي ثانية (5 ثواني)
    // ولما ترن.. نادي على "mom"
    Timer kitchenTimer = new Timer(5000, mom);
    
    // 3. شغل المنبه
    kitchenTimer.start();
    ```
    

### الخلاصة عشان الطفل الصغير يفهم:

- الـ **`Timer`**: هو منبه بيعد وقت.
    
- الـ **`ActionListener`**: هو الشخص اللي قاعد مستني المنبه يرن عشان يعمل حاجة.
    
- الـ **`actionPerformed`**: هو الفعل اللي الشخص هيعمله (يقفل النار، يطبع رسالة، يزمر).
    

كده "عمو Timer" بقى صاحبنا وفهمنا هو مين؟ ولا لسه غامض؟
